# The demo worked. It still never shipped.

`2026 June 3`

There is a specific place AI projects go to die, and it is not the lab. It is the gap between a demonstration that impressed everyone in the room and a system that runs every day without someone watching it. The [pilot purgatory study](disclaimer.md) names the pattern directly: the proof-of-concept works because a person curates the inputs, handles the edge cases by hand, and quietly retries when it fails. Production has none of that grace. It meets the messy invoice, the half-finished record, the customer who phrases the question in a way no one anticipated. The demo was a performance. Production is a witness.

This is the mechanism behind the numbers. We already noted that [only a small minority of organisations scale AI](2026-05-31-ai-adoption-reality.md) while the rest run pilots forever, and that industrial surveys keep [finding firms stuck at the same stage year after year](disclaimer.md). The reason is rarely the model. It is that no one budgeted for the unglamorous work the demo hid: error handling, monitoring, the human who checks the output, the retraining when the data drifts. The [permanent beta](disclaimer.md) concept captures the trap at the far end — a system that is always almost ready and never trusted enough to rest on.

The practical lesson is to treat the demo as the cheapest part of the project, not the proof it will work. The question that separates a pilot from a deployment is not "did it work once" but "what happens the day it doesn't, and who notices."
